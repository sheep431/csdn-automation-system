# CSDN 自主运营系统 7 层路线图

> 目标：把 CSDN 账号运营从顶层设计逐步推进到可执行、可追踪、可持续优化的落地系统。

## 总原则

- 先定义顶层，再往下拆。
- 每一层只解决自己的问题，不越层乱做。
- 每层都要有“输入、输出、验收标准”。
- 先人工跑通，再把重复部分自动化。
- 人审关键节点不取消。

## 七层定义

### 第 0 层：经营输入与决策层

解决什么：账号数据、用户反馈、外部参考、销售数据怎么进入决策系统，并持续学习进化。

输出物：
- 账号数据采集清单
- 用户反馈归档方式
- 对标账号与优秀案例记录
- 专栏转化跟踪表
- 可执行的决策结论

验收标准：
- 能持续收集并归档经营输入
- 能把输入转成可执行决策
- 能支持选题、发布和变现策略持续进化

---

### 第 1 层：战略层

解决什么：为什么做、做成什么样、双账号各自扮演什么角色。

输出物：
- 项目目标
- 账号分工
- 内容方向原则
- 成功标准
- 不做什么

验收标准：
- 任何人看到都能理解项目的边界和目标
- 两个账号的角色差异清晰

---

### 第 2 层：业务层

解决什么：写什么、给谁看、为什么值、什么题更适合哪个账号。

输出物：
- 选题方向
- 专栏策略
- 受众画像
- 内容类型分布
- 账号适配规则

验收标准：
- 能把题目分到正确账号
- 能判断题目值不值得写

---

### 第 3 层：流程层

解决什么：事情怎么流转，从选题到发布如何推进。

输出物：
- 选题 -> 草稿 -> 审核 -> 发布 -> 反馈 流程
- 每一步的输入输出
- 每一步的责任人/系统职责

验收标准：
- 每个任务都有明确状态
- 任何阶段都能知道下一步是什么

---

### 第 4 层：规则层

解决什么：每一步按什么标准判断、通过或退回。

输出物：
- 选题评分规则
- 草稿合格标准
- 审核通过标准
- 退回修改标准
- 优先级规则

验收标准：
- 判断标准尽量统一
- 人工决策不靠临场感觉

---

### 第 5 层：数据与状态层

解决什么：系统怎么记录对象、状态怎么变化、历史怎么追踪。

输出物：
- 选题对象
- 草稿对象
- 审核包对象
- 发布任务对象
- 反馈对象
- 状态流转定义

验收标准：
- 任意任务都能追踪到当前状态和历史记录
- 数据结构可支撑后续自动化

---

### 第 6 层：执行层

解决什么：具体怎么落到代码、命令、自动化脚本和通知。

输出物：
- CLI 命令
- 自动化脚本
- 数据目录
- 日志与错误处理
- 通知方式

验收标准：
- 能日常运行
- 能逐步替换人工重复操作

## 递进顺序

推荐顺序如下：

1. 经营输入层先建立
2. 战略层定清楚
3. 业务层把题和账号关系定清楚
4. 流程层把工作怎么走定清楚
5. 规则层把怎么判断定清楚
6. 数据层把怎么记录定清楚
7. 执行层再把它映射到 automation-system

## 当前推进策略

- 先完成第 0 层和第 1 层
- 再把业务、流程和规则补齐
- 最后映射到现有代码结构
