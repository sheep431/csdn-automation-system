# CSDN 自主运营系统 - 第 1 层：战略层规范

> 这一层只回答三件事：为什么做、做成什么样、两个账号分别承担什么任务。

## 1. 这一层的职责

战略层是整个系统的顶层约束。它决定后续所有内容、流程、规则、数据和代码应该服务什么目标。

这一层不讨论具体怎么写文章，不讨论数据表怎么建，不讨论 CLI 怎么设计，只定义方向和边界。

## 2. 核心目标

### 2.1 总目标

把 CSDN 双账号运营做成一个可持续、可审稿、可追踪、可逐步自动化的日常工作流。

### 2.2 过程目标

- 保持稳定产出
- 保留人审关键节点
- 降低每日重复劳动
- 逐步把经验沉淀成系统规则
- 让选题、写作、审核、发布形成闭环

### 2.3 结果目标

- 让新号承担主增长任务
- 让旧号承担辅助输出和经验型内容
- 让每篇内容都有来源、归属、审核与反馈记录

## 3. 双账号分工

### 3.1 新号：技术小甜甜

定位：主增长账号。

适合的内容：
- 具体实操
- 问题解决
- 教程类
- 一步一步能做出来的内容
- 面向新读者的清晰表达

这个账号的风格应该更直接、更实用、更少铺垫。

### 3.2 旧号：踏雪无痕老爷子

定位：辅助增长和稳态输出账号。

适合的内容：
- 经验总结
- 方法论
- 判断与观点
- 复盘
- 更成熟、更有“为什么”的表达

这个账号的风格应该更像“经验老手给结论”，而不是“新手手把手教程”。

## 4. 战略原则

### 4.1 新号优先

当前系统的重点是新号，不把老号作为主突破口。

### 4.2 人审优先

文章是否值得发、是否适配账号、是否符合当前语气和风格，这些判断优先交给人。

### 4.3 先跑通闭环，再谈规模化

先把单篇、单日、单账号流程跑通，再扩展到更高频和更多自动化。

### 4.4 不依赖旧 markdown 作为生产主输入

历史 markdown 只作为背景参考，不作为批量生产的主来源。

### 4.5 默认按项目目录沉淀资产

所有策略、规范、选题、复盘、运行记录都尽量留在统一项目目录里，避免跨目录混乱。

## 5. 成功标准

如果战略层做对了，应满足以下判断：

- 任何一个选题都能被解释为什么属于这个账号
- 任何一篇文章都知道它要服务什么目标
- 任何一次人工判断都有明确依据
- 任何一次发布或退回都能被追踪

## 6. 不做什么

战略层明确不做以下事情：

- 不追求一开始就全自动直发
- 不把批量旧 markdown 当现成产能
- 不让流程依赖临时口头约定
- 不把两个账号混成一个内容池
- 不让系统在顶层目标未明确时就进入细节实现

## 7. 这一层的输出物

这一层完成后，应输出以下内容：

- 项目总目标
- 账号分工说明
- 内容方向原则
- 生产节奏原则
- 不做事项列表

## 8. 下一层接口

战略层完成后，下一层需要回答：

- 什么内容适合哪个账号
- 什么内容值得写
- 内容从哪里来
- 如何把题目和账号匹配起来

也就是进入第 2 层：业务层。
